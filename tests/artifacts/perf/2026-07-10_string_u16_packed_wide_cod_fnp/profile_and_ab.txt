LEDGER-FIRST ROUTING
====================
The complex128 dense setops parity/grid family, structured (int/uint width
1/2/4/8 + float32/float64) bitplane family, and narrow Latin-1 U/S packed-u64
unique/setops family were already shipped on main. This attempt did not reopen
the ledger-closed dense row-unique occupancy/rank table, U8 union-only helper,
float radix-select median, or float16-via-f32 sort ideas. The open residual was
Latin-1 fixed-width Unicode U9..U16, which still used record-index memcmp sort.

Mechanism selected from ranks 1-2: replace 36..64-byte cache-cold record
comparisons with one lexicographic (high u64, low u64, original u32 index) key
sort. Packing is big-endian per Latin-1 codepoint, so derived tuple Ord is the
same order as the existing record comparator. Non-Latin-1, U17+, all S widths,
and every unsupported layout keep the existing fallback. No unsafe and no C
BLAS/LAPACK/XLA were added.
