# franken_numpy-29f7f - zero-based dense sort no-ship proof

## Target

- Bead: `franken_numpy-29f7f`
- Hotspot: `core_ops/sort_quicksort_1m`
- Profile-backed baseline: rch Criterion on `vmi1156319` after `dfbdd3eb`
- Baseline: `8.7001 ms` center, CI `[8.6091 ms, 8.8439 ms]`

The benchmark input is an exact zero-based integer permutation generated by
`(i * 48271) % 1_000_000`. The tested lever added a one-pass bitmap detector for
exact finite values in `0..len`, emitting sorted `0..len` directly and falling
through for duplicate, NaN, signed-zero, non-integer, negative, out-of-range, and
non-zero-based inputs.

## Decision

Rejected. The first candidate measurement used a different worker from the
baseline and still overlapped the baseline confidence interval. The same-worker
rerun for the candidate regressed. Score: Impact `1` x Confidence `1` / Effort
`1` = `1.0`, below the required `2.0` keep threshold.

No source change is kept for this bead. Observable behavior is unchanged by the
closeout: ordering, tie-breaking, floating-point edge cases, negative zero,
NaN fallback behavior, and RNG state remain on the pre-existing implementation.
Public API golden file sha256 remains
`1caa4743eb3f8f97cfc6bb08e373d14c17eafb2573583c2d8b9c2847d7bc0ad1`.
