## Upgrade workflow (when numpy version bumps)

1. Pin the new numpy version via `FNP_ORACLE_PYTHON=/path/to/python3.x`.
2. Run `cargo run --bin regenerate_all_fixtures -p fnp-conformance`.
3. Diff-review each `captured` fixture: `git diff fixtures/*_differential_cases.json`.
4. For each unexpected divergence, either (a) accept — bump numpy version in `fixtures/PROVENANCE.md`, or (b) file a DISC-NNN in `DISCREPANCIES.md` and XFAIL the affected test.
5. Manual fixtures: visually inspect on numpy releases that ship breaking ufunc/ma semantics changes.
6. Update `fixtures/PROVENANCE.md` `Reference NumPy` line and each captured fixture's `Last regen` column.
